# OpenClaw 安装程序 — 可行性与完整性测试指南

本文档说明如何系统化地验证安装程序的**可行性**（能否跑通）和**完整性**（功能是否齐全、边界是否覆盖）。

---

## 一、快速验证（约 5 分钟）

在项目目录下执行：

```powershell
cd "E:\公司包包\openclaw行程文件\openclaw安装程序"
python test_installer.py
```

脚本将自动检查：

- Python 语法与导入（含 tkinter）
- 配置生成逻辑（模型列表、API Key 映射、JSON 结构）
- 安装目录与配置文件路径解析

全部通过即表示**基础可行性**正常。

---

## 二、可行性测试（程序能否跑通）

### 2.1 环境与依赖

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | `python --version` | 显示 3.8 及以上 |
| 2 | `python -c "import tkinter; print('tk ok')"` | 输出 `tk ok` |
| 3 | 运行 `python test_installer.py` | 所有检查通过 |

### 2.2 程序能正常启动

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 双击 `安装OpenClaw.bat` 或执行 `python openclaw_installer.py` | 主窗口弹出，无报错 |
| 2 | 点击「仅配置模型」 | 配置窗口弹出，可选模型、输入 API Key |
| 3 | 选择任意模型，输入任意字符串作为 API Key，点「保存配置」 | 提示保存成功，并给出路径 |

保存成功后，检查 `C:\Users\<你的用户名>\.openclaw\openclaw.json` 是否存在且内容与所选模型、API Key 一致 → 说明**配置流程完整可行**。

### 2.3 完整安装流程（需网络，约 10–20 分钟）

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1 | 若本机**未安装** Node.js：点「开始安装」 | 先下载/安装 Node.js，再安装 OpenClaw；若已安装则跳过 Node.js |
| 2 | 安装完成后弹出配置窗口 | 完成模型与 API Key 配置并保存 |
| 3 | 新开 PowerShell，执行 `openclaw --version` | 能输出版本号 |

满足上述即说明**从零到可用的安装链路可行**。

## 三、完整性测试（功能与边界是否齐全）

可结合 `测试任务模板.md` 做勾选式检查。

### 3.1 功能完整性

| 编号 | 测试项 | 操作要点 | 通过标准 |
|------|--------|----------|----------|
| F1 | Node.js 检测 | 有 Node 时运行安装程序 | 日志显示「Node.js 已安装」，不下载 |
| F2 | Node.js 安装 | 无 Node 时运行「开始安装」 | 下载 MSI、安装、日志有成功提示 |
| F3 | OpenClaw 安装 | 「开始安装」跑完 | 日志有「OpenClaw 安装成功」 |
| F4 | 国产模型配置 | 配置窗口选「国产模型」→ 任选模型 → 填 Key → 保存 | 生成 `openclaw.json`，`model` 与所选一致 |
| F5 | 国际模型配置 | 同上，选「国际模型」 | 同上 |
| F6 | Thinking 模式 | 勾选「启用 Thinking 模式」后保存 | JSON 中 `"thinking": true` |
| F7 | WebChat 开关 | 取消「启用 WebChat」后保存 | JSON 中 `webchat.enabled: false` |
| F8 | API Key 显示/隐藏 | 点击「显示」「隐藏」 | 输入框在明文与 `*` 间切换 |
| F9 | 测试连接 | 配置好后点「测试连接」 | 弹出配置摘要（模型、提供商、Key 脱敏） |
| F10 | 仅配置模型 | 已安装 OpenClaw 时只点「仅配置模型」 | 不重装，只写配置 |

### 3.2 边界与异常

| 编号 | 测试项 | 操作要点 | 通过标准 |
|------|--------|----------|----------|
| B1 | 未选模型保存 | 不选模型，只填 API Key，点保存 | 提示「请选择模型」 |
| B2 | 未填 API Key 保存 | 选模型但不填 Key，点保存 | 提示「请输入 API Key」 |
| B3 | 网络异常 | 断网或断代理后「开始安装」（无 Node 时） | 有错误提示，并提示可手动安装 Node |
| B4 | 配置目录无写权限 | 对 `~/.openclaw` 设只读或无权限（需提前改权限） | 保存时提示失败，不崩溃 |

### 3.3 兼容性（可选）

| 项目 | 建议验证 |
|------|----------|
| Windows 10 / 11 | 各在一台机器或虚拟机跑一遍「快速验证 + 程序启动 + 仅配置模型」 |
| Python 3.8 / 3.10 / 3.12 | 用不同 Python 运行 `test_installer.py` 和 `openclaw_installer.py` |

## 四、测试结果记录

建议在 `测试报告_v1.0.md` 或新建一轮报告中记录：

- **可行性**：快速验证 ✓/✗，程序启动 ✓/✗，完整安装 ✓/✗（可注明环境：有无 Node、网络情况）。
- **完整性**：F1–F10、B1–B4 的通过情况；若有失败，写明步骤与现象。
- **结论**：是否达到「可发布」或「需修复后再测」（可引用测试报告中的改进建议）。

---

## 五、推荐测试顺序

1. 运行 `python test_installer.py` → 确认基础可行。
2. 启动主程序 + 「仅配置模型」+ 保存并检查 JSON → 确认配置功能完整。
3. 在无 Node 环境下跑一遍完整安装（可选，需网络）→ 确认安装链路可行。
4. 按 F1–F10、B1–B4 做勾选 → 确认完整性。
5. 将结果填入测试报告，给出可行性与完整性结论。

按上述顺序执行，即可系统化地测试安装程序的可行性与完整性。
