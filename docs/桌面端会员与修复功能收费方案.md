# ButlerClaw 桌面端会员与修复功能收费方案

## 1. 目标

- 与运营平台（ButlerClaw Hub）**会员体系一致**：免费版（free）、专业版（pro）。
- **修复功能收费**：诊断免费，自动修复（单条修复 + 一键修复）仅专业版可用。
- 桌面端通过登录 Hub 账号获取会员状态，按会员等级控制功能入口与执行。

---

## 2. 与运营平台一致的会员机制

### 2.1 会员定义（与 Hub 对齐）

| 等级 | 标识 | 说明 |
|------|------|------|
| 免费版 | `free` | 诊断、技能市场浏览与免费技能安装、基础功能 |
| 专业版 | `pro` | 免费版 + **智能修复**（一键修复、单条修复）、付费技能权益、优先支持等 |

数据来源：Hub 用户表 `users.membershipLevel`、`users.membershipExpireAt`（专业版到期时间）。

### 2.2 桌面端需要的信息

从运营平台拉取并缓存的字段建议为：

- `membershipLevel`: `"free"` | `"pro"`
- `membershipExpireAt`: ISO 日期字符串或 `null`（专业版到期日，过期后按 free 处理）

可选一并缓存：`userId`、`username`，用于展示「已登录为 xxx · 专业版」等。

---

## 3. 桌面端与 Hub 的对接方式

### 3.1 登录与 Token

- **登录入口**：桌面端提供「登录 ButlerClaw 账号」入口（如设置页、诊断页顶部），打开运营平台登录页或内嵌浏览器完成登录，登录成功后由 Hub 将 token 回传桌面端（或桌面端从本地/临时页获取）。
- **Token 存储**：在本地配置中持久化 Hub 登录态，例如：
  - 路径：与现有 `~/.openclaw/openclaw.json` 同目录，单独文件如 `~/.openclaw/hub.json`，或在同一 JSON 中增加 `hub` 节点。
  - 内容建议：`{ "token": "JWT...", "userId": "...", "username": "...", "membershipLevel": "free"|"pro", "membershipExpireAt": "..."|null, "updatedAt": "ISO时间" }`
- **安全**：文件权限建议仅当前用户可读（如 0o600），不在日志中打印 token。

### 3.2 获取会员信息（与运营平台一致）

- **接口**：使用运营平台现有能力即可。
  - 方式 A：直接请求 Hub 前端域名下的接口（需 Hub 支持 CORS 或提供桌面用域名），例如 `GET https://<hub域名>/api/auth/me`，请求头 `Authorization: Bearer <token>`，返回 body 中含 `membershipLevel`、`membershipExpireAt` 等。
  - 方式 B：Hub 提供单独的「桌面端 API」域名或路径，返回同一套用户与会员信息。
- **桌面端调用时机**：
  - 启动时若存在 token，先读本地缓存并展示会员状态，再异步请求 `/api/auth/me` 刷新会员信息并写回缓存。
  - 进入「诊断」或点击「修复」前，若缓存已过期（如超过 5–10 分钟），可再请求一次以尽量保证与运营平台一致。
- **过期与离线**：
  - `membershipExpireAt` 已过：桌面端视为 `free`，修复功能不可用。
  - 网络失败或 401：若本地有缓存则沿用上次结果并标记「可能已过期」；若无缓存则视为未登录，修复功能不可用。

### 3.3 功能与会员对应关系

| 功能 | 免费版 | 专业版 |
|------|--------|--------|
| 诊断（开始诊断、查看结果、导出报告） | ✅ | ✅ |
| 单条「修复」按钮（执行该条自动修复） | ❌ 提示升级 | ✅ |
| 一键修复（批量自动修复） | ❌ 提示升级 | ✅ |
| 技能市场浏览与免费技能安装 | ✅ | ✅ |
| 付费技能下载/安装 | 按 Hub 规则（可要求已登录） | ✅ |

「修复」包括：`diagnose.smart_fix` 的自动修复、以及诊断面板中每条结果上的「修复」按钮与「一键修复」入口。

## 4. 修复功能收费的具体实现要点

### 4.1 入口与 UI

- **单条「修复」按钮**（诊断结果表格内）  
  - 未登录或会员非 pro：点击后提示「自动修复为专业版功能，请登录并升级」并引导至 Hub 登录/定价页。  
  - 已登录且为 pro（未过期）：执行现有 `fix_action`/智能修复逻辑。
- **一键修复按钮**  
  - 未登录或会员非 pro：点击后同样提示专业版并引导至 Hub。  
  - 已登录且为 pro：执行现有 `_auto_fix` 流程（含 `SmartFixAssistant.analyze_and_fix` 等）。

### 4.2 执行前校验

- 在真正调用 `SmartFixAssistant.analyze_and_fix` 或执行任意自动修复前，统一做一次「当前是否为 pro」校验（读本地缓存 + 必要时请求 Hub）。
- 若为 pro 但 `membershipExpireAt` 已过：按 free 处理，不执行修复并提示续费。

### 4.3 提示与引导

- 未登录：提示「登录 ButlerClaw 账号后可使用智能修复（专业版）」+ 按钮「去登录」跳转 Hub 登录页。
- 已登录为 free：提示「智能修复为专业版功能，升级后可一键修复」+ 按钮「查看价格」跳转 Hub 定价页。
- 专业版已过期：提示「您的专业版已过期，续费后继续使用智能修复」+ 同上定价页。

Hub 定价页路径示例：`https://<hub域名>/pricing`，登录页：`https://<hub域名>/login`。

## 5. 配置与代码建议

### 5.1 本地配置结构示例

在 `~/.openclaw/` 下增加 `hub.json`（或合并进现有配置的 `hub` 节点）：

```json
{
  "token": "eyJ...",
  "userId": "xxx",
  "username": "用户昵称",
  "membershipLevel": "pro",
  "membershipExpireAt": "2026-12-31T23:59:59.000Z",
  "updatedAt": "2026-03-14T12:00:00.000Z"
}
```

- 仅当用户曾在桌面端完成「登录 ButlerClaw 账号」并成功拉取到会员信息后写入。
- `updatedAt` 用于本地判断是否需重新请求 `/api/auth/me`（如超过 5 分钟则刷新）。

### 5.2 Hub 基础 URL

- 建议可配置：如 `openclaw.json` 或 `hub.json` 中增加 `hubBaseUrl`（默认可为运营平台生产域名），便于开发/测试环境切换。
- 桌面端请求会员信息：`GET {hubBaseUrl}/api/auth/me`，头 `Authorization: Bearer {token}`。

### 5.3 代码层

- **会员模块**：新增一模块（如 `hub_auth.py` 或 `membership.py`），职责包括：读/写本地 hub 配置、调用 Hub `/api/auth/me`、解析 `membershipLevel` 与 `membershipExpireAt`、提供 `is_pro()` / `can_use_fix()` 等接口。
- **诊断/修复处**：在 `openclaw_assistant.py` 中，单条「修复」与「一键修复」入口处先调 `can_use_fix()`；若为 False，弹出上述提示并 return，不执行修复逻辑。
- **登录入口**：在设置或诊断面板提供「登录 ButlerClaw 账号」；登录成功后由 Hub 或本地回调写入 token，再请求一次 `/api/auth/me` 并写入 `hub.json`。

## 6. 与运营平台的数据流小结

```
桌面端                          运营平台 (Hub)
   |                                  |
   |  GET /api/auth/me                |
   |  Authorization: Bearer <token>   |
   | --------------------------------->
   |                                  | 校验 JWT，查 users 表
   |  200 { membershipLevel,          |
   |        membershipExpireAt, ... } |
   | <---------------------------------
   |                                  |
   |  桌面端缓存并判断 free/pro       |
   |  修复入口根据 pro 决定是否执行   |
```

- 会员定义、等级、到期时间均以运营平台为准；桌面端只读并缓存，不做修改。
- 付费技能购买、续费等均在 Hub 完成；桌面端仅根据返回的 `membershipLevel`、`membershipExpireAt` 控制修复功能与提示。

---

## 7. 实施顺序建议

1. **配置与会员模块**：实现 `hub.json` 读/写、`GET /api/auth/me` 调用、`is_pro()` / `can_use_fix()`。
2. **修复入口鉴权**：在单条「修复」与「一键修复」处加 `can_use_fix()` 判断及提示/跳转。
3. **登录入口**：在桌面端增加「登录 ButlerClaw 账号」流程，完成 token 写入与首次会员拉取。
4. **文案与链接**：统一「专业版」「升级」「续费」等文案，并确保 Hub 定价/登录页链接可配置。

按此方案可实现：**修复功能收费**，且桌面端会员与运营平台**一致以 free/pro 为准**，后续若 Hub 增加更多专业版权益，桌面端只需根据 `membershipLevel` 或新字段扩展判断即可。
