## 二、技术选型

### 2.1 前端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| **Next.js** | 14.x | React 框架，支持 SSR/SSG |
| **TypeScript** | 5.x | 类型安全 |
| **TailwindCSS** | 3.x | 原子化 CSS |
| **shadcn/ui** | latest | UI 组件库 |
| **TanStack Query** | 5.x | 数据获取和缓存 |
| **Zustand** | 4.x | 状态管理 |
| **React Hook Form** | 7.x | 表单处理 |
| **Zod** | 3.x | 数据校验 |

**选择理由**：
- Next.js：SEO友好、性能优秀、生态丰富
- TypeScript：大型项目必备，减少运行时错误
- TailwindCSS：开发效率高，易于维护
- shadcn/ui：高质量组件，可定制性强

### 2.2 后端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| **Go** | 1.21+ | 主服务开发 |
| **Gin** | 1.9+ | Web 框架 |
| **GORM** | 1.25+ | ORM |
| **Node.js** | 20.x | 部分服务（如实时通信）|
| **Express** | 4.x | Node Web 框架 |

**选择理由**：
- Go：高性能、并发能力强、部署简单
- Node.js：适合 I/O 密集型任务（如实时通知）

### 2.3 数据库

| 数据库 | 用途 | 版本 |
|--------|------|------|
| **MySQL 8.0** | 主数据库 | 8.0+ |
| **Redis 7** | 缓存/会话/限流 | 7.x |
| **Elasticsearch 8** | 搜索 | 8.x |
| **MongoDB 7** | 日志/文档 | 7.x |

**选择理由**：
- MySQL：成熟稳定，事务支持好
- Redis：高性能缓存，支持多种数据结构
- ES：强大的全文搜索能力
- MongoDB：灵活的文档存储，适合日志

### 2.4 基础设施

| 技术 | 用途 |
|------|------|
| **Docker** | 容器化 |
| **Kubernetes** | 容器编排 |
| **Nginx** | 反向代理/负载均衡 |

| **Prometheus + Grafana** | 监控告警 |
| **ELK Stack** | 日志分析 |

## 三、核心服务设计

### 3.1 用户服务 (User Service)

**职责**：用户注册、登录、权限管理

**API**：
```go
// 注册
POST /api/v1/auth/register
{
    "username": "string",
    "email": "string",
    "password": "string"
}

// 登录
POST /api/v1/auth/login
{
    "email": "string",
    "password": "string"
}

// 获取用户信息
GET /api/v1/users/:id

// 更新用户信息
PUT /api/v1/users/:id
```

**数据库表**：
```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    avatar VARCHAR(255),
    role ENUM('user', 'developer', 'admin') DEFAULT 'user',
    status ENUM('active', 'inactive', 'banned') DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_email (email),
    INDEX idx_username (username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.2 技能服务 (Skill Service)

**职责**：技能 CRUD、审核、搜索

**API**：
```go
// 获取技能列表
GET /api/v1/skills?page=1&size=20&category=tool&sort=downloads

// 获取技能详情
GET /api/v1/skills/:id

// 上传技能（开发者）
POST /api/v1/skills
Content-Type: multipart/form-data
{
    "name": "string",
    "description": "string",
    "category": "string",
    "tags": ["string"],
    "price": 0,
    "file": File
}

// 更新技能
PUT /api/v1/skills/:id

// 删除技能
DELETE /api/v1/skills/:id

// 搜索技能
GET /api/v1/skills/search?q=keyword&page=1&size=20
```

**数据库表**：
```sql
CREATE TABLE skills (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    skill_id VARCHAR(100) UNIQUE NOT NULL,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    author_id BIGINT NOT NULL,
    category VARCHAR(50),
    tags JSON,
    icon VARCHAR(255),
    screenshots JSON,
    version VARCHAR(20),
    download_url VARCHAR(255),
    readme TEXT,
    price DECIMAL(10, 2) DEFAULT 0,
    pricing_model ENUM('free', 'onetime', 'subscription') DEFAULT 'free',
    status ENUM('pending', 'approved', 'rejected', 'disabled') DEFAULT 'pending',
    downloads INT DEFAULT 0,
    rating DECIMAL(2, 1) DEFAULT 5.0,
    rating_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (author_id) REFERENCES users(id),
    INDEX idx_category (category),
    INDEX idx_status (status),
    INDEX idx_price (price),
    FULLTEXT INDEX idx_search (name, description)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.3 会员服务 (Membership Service)

**职责**：会员订阅、权益管理、过期处理

**API**：
```go
// 获取会员方案
GET /api/v1/membership/plans

// 订阅会员
POST /api/v1/membership/subscribe
{
    "plan": "basic|premium|developer",
    "duration": "month|year"
}

// 获取会员状态
GET /api/v1/membership/status

// 取消订阅
POST /api/v1/membership/cancel
```

**数据库表**：
```sql
CREATE TABLE membership_plans (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    code VARCHAR(20) UNIQUE NOT NULL,
    description TEXT,
    price_monthly DECIMAL(10, 2) NOT NULL,
    price_yearly DECIMAL(10, 2) NOT NULL,
    benefits JSON,
    status ENUM('active', 'inactive') DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE user_memberships (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    plan_id BIGINT NOT NULL,
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    status ENUM('active', 'expired', 'cancelled') DEFAULT 'active',
    auto_renew BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (plan_id) REFERENCES membership_plans(id),
    INDEX idx_user (user_id),
    INDEX idx_end_date (end_date)
);
```

### 4.2 容器化部署

**Dockerfile 示例**：
```dockerfile
# 构建阶段
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o main .

# 运行阶段
FROM alpine:latest
RUN apk --no-cache add ca-certificates
WORKDIR /root/
COPY --from=builder /app/main .
EXPOSE 8080
CMD ["./main"]
```

**Kubernetes 部署示例**：
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: user-service
spec:
  replicas: 3

        app: user-service
    spec:
      containers:
      - name: user-service
        image: openclaw/user-service:latest
        ports:
        - containerPort: 8080
        env:
        - name: DB_HOST

            secretKeyRef:
              name: db-secret
              key: host
```

## 五、安全设计

### 5.1 认证授权

- **JWT Token**：无状态认证，支持过期和刷新
- **OAuth 2.0**：支持第三方登录（GitHub/Google）
- **RBAC**：基于角色的权限控制

### 5.2 数据安全

- **传输加密**：全站 HTTPS
- **存储加密**：敏感数据加密存储
- **密码安全**：bcrypt 哈希，加盐

### 5.3 防护机制

- **Rate Limiting**：API 限流，防刷
- **WAF**：Web 应用防火墙
- **DDoS 防护**：CDN 层防护
- **SQL 注入**：参数化查询
- **XSS 防护**：输入过滤，CSP
