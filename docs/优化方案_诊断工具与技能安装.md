# OpenClaw Butler 诊断工具与技能安装优化方案

## 当前问题分析

### 1. 诊断工具 (DiagnosePanel)

**现状**:
- 8x2 诊断区块矩阵（环境、Gateway、渠道、模型、配置、状态、工具、自动化）
- 诊断项目总览表格（诊断项、状态、建议、操作）
- 一键修复功能（基础版，部分功能标记为"专业版"）
- 环境自检功能
- 导出诊断报告

**存在的问题**:
1. **诊断结果展示不够直观** - 表格形式，信息密度高但可读性一般
2. **修复功能有限** - 大部分修复标记为"专业版"，实际可用功能少
3. **缺乏历史记录** - 无法对比多次诊断结果
4. **缺少可视化图表** - 没有健康度评分、趋势图等
5. **一键修复不够智能** - 只能修复基础问题，复杂问题仍需手动处理

### 2. 技能安装 (SkillsPanelNew)

**现状**:
- 左右分栏布局：列表 + 详情
- 已安装 / 可安装分区显示
- 搜索 + 分类筛选
- 技能详情面板

**存在的问题**:
1. **安装失败处理不完善** - 网络问题、权限问题等没有明确提示
2. **缺少批量操作** - 无法批量安装/更新/卸载
3. **没有安装队列** - 多个技能同时安装时没有进度管理
4. **缺少依赖检查** - 安装前没有检查前置依赖
5. **没有离线安装支持** - 无法从本地文件安装

```python
    """技能安装队列管理器"""
    
    def __init__(self):
        self.queue = []

    def add_to_queue(self, skill_id, priority=0):
        """添加到安装队列"""
        task = {

            "status": "pending",  # pending, downloading, installing, completed, failed
            "priority": priority,
            "progress": 0,
            "error": None,
            "start_time": None,
            "end_time": None
        }
        self.queue.append(task)
        self._sort_queue()
        return task
    
    def batch_install(self, skill_ids):
        """批量安装技能"""
        for skill_id in skill_ids:
            self.add_to_queue(skill_id)
        self._process_queue()
    
    def _process_queue(self):
        """处理队列"""
        # 串行或并行（可配置）处理安装任务
        pass
    
    def get_progress(self):
        """获取整体进度"""
        total = len(self.queue)
        completed = sum(1 for t in self.queue if t["status"] in ("completed", "failed"))
        return completed / total if total > 0 else 0
```

```python
class DependencyChecker:
    """依赖检查器"""
    
    def check_before_install(self, skill_id):
        """安装前检查依赖"""
        requirements = self._get_skill_requirements(skill_id)
        
        checks = {
            "node_version": self._check_node_version(requirements.get("node")),
            "npm_packages": self._check_npm_packages(requirements.get("packages", [])),
            "system_bins": self._check_system_bins(requirements.get("bins", [])),
            "env_vars": self._check_env_vars(requirements.get("env", [])),
            "disk_space": self._check_disk_space(requirements.get("size", 0)),
        }
        
        missing = [k for k, v in checks.items() if not v["ok"]]
        
        return {
            "can_install": len(missing) == 0,
            "checks": checks,
            "missing": missing,
            "auto_fixable": self._get_auto_fixable(missing)
        }
    
    def auto_fix_dependencies(self, missing):
        """自动修复依赖"""
        for item in missing:
            if item["auto_fixable"]:
                self._fix_dependency(item)
```

#### 4. 安装失败智能处理

```python
class InstallErrorHandler:
    """安装错误处理器"""
    
    ERROR_PATTERNS = {
        "rate_limit": {
            "patterns": ["rate limit exceeded", "403"],
            "message": "GitHub API 速率限制，建议：",
            "solutions": [
                "等待1小时后重试",
                "配置 GitHub Token 提高限额",
                "使用代理访问"
            ]
        },
        "network_error": {
            "patterns": ["connection refused", "timeout", "reset"],
            "message": "网络连接问题，建议：",
            "solutions": [
                "检查网络连接",
                "配置代理",
                "使用离线安装"
            ]
        },
        "permission_denied": {
            "patterns": ["permission denied", "EACCES"],
            "message": "权限不足，建议：",
            "solutions": [
                "以管理员身份运行",
                "修改目录权限",
                "使用 nvm 管理 Node.js"
            ]
        },
        "skill_not_found": {
            "patterns": ["not found", "404"],
            "message": "技能不存在，建议：",
            "solutions": [
                "检查技能名称拼写",
                "搜索正确的技能ID",
                "查看ClawHub上的可用技能"
            ]
        }
    }
    
    def analyze_error(self, error_output):
        """分析错误输出"""
        for error_type, info in self.ERROR_PATTERNS.items():
            for pattern in info["patterns"]:
                if pattern in error_output.lower():
                    return {
                        "type": error_type,
                        "message": info["message"],
                        "solutions": info["solutions"]
                    }
        return None
    
    def get_retry_strategy(self, error_type):
        """获取重试策略"""
        strategies = {
            "rate_limit": {"retry": True, "delay": 3600, "max_retries": 3},
            "network_error": {"retry": True, "delay": 30, "max_retries": 5},
            "permission_denied": {"retry": False, "manual_fix": True},
            "skill_not_found": {"retry": False, "search_alternative": True}
        }
        return strategies.get(error_type, {"retry": False})
```

**UI 展示**:
- 错误弹窗显示问题原因和解决方案
- "重试"按钮（根据错误类型决定是否显示）
- "查看详细日志"按钮
